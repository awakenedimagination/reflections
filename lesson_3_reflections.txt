Q. When would you want to create a remote repository rather than keeping all your work local?

A. When you're sharing with someone or collaborating.

Q. Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

A. Because you may be testing changes or not be ready to either show the public or copy onto your working directory.

Q. Describe the differences between forks, clones, and branches. When would you use one instead of another?

A. A fork is a clone of someone's repository on github servers. A clone is a local clone or a clone of a repository onto your local system. A branch is new label for separate set of commits.
